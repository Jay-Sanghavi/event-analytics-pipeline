ARCHITECTURE SUMMARY
================================================================================

Data Pipeline Layers:
  Bronze Layer:   Raw JSON Lines → Parquet (year/month/day/hour partitions)
  Silver Layer:   Cleaned, deduplicated events (daily partitions)
  Gold Layer:     5 pre-aggregated analytical tables

Storage:
  Source Bucket:    capstone-events-jsanghvi-{ACCOUNT}
  Analytics Bucket: capstone-analytics-jsanghvi-{ACCOUNT}

Databases:
  bronze_jsanghvi   (raw events)
  silver_jsanghvi   (cleaned events)
  gold_jsanghvi     (5 analytical tables)

ETL Processing:
  Tool: AWS Glue (PySpark)
  Job Name: capstone-etl-jsanghvi
  Processing: Incremental via Glue bookmarks (no reprocessing)
  Schedule: Manual (or can be scheduled via EventBridge)

Query Engine:
  Athena (standard SQL)
  Query Latency: <1 second
  Cost: ~$0.001 per query

DEPLOYMENT QUICK START
================================================================================

1. Deploy CloudFormation (5-10 minutes):
   aws cloudformation create-stack \
     --stack-name capstone-jsanghvi \
     --template-body file://capstone-starter.cfn.yaml \
     --parameters ParameterKey=StudentId,ParameterValue=jsanghvi \
     --region us-west-2

2. Wait for Event Generation (5+ minutes)

3. Run Build Pipeline Notebook (5 minutes):
   jupyter notebook build_pipeline.ipynb

4. Execute Glue Job (5-15 minutes):
   aws glue start-job-run \
     --job-name capstone-etl-jsanghvi \
     --region us-west-2

5. Test Queries (1 minute):
   SELECT * FROM gold_jsanghvi.conversion_funnel LIMIT 5;
   SELECT * FROM gold_jsanghvi.hourly_revenue LIMIT 5;
   -- ... all 5 queries from queries.sql

Total Time: ~40 minutes
